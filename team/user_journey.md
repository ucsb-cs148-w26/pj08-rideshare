# User Journey

This document outlines the typical user journeys for the UCSB Rideshare App, focusing on the primary user roles: Riders and Drivers.

---

## User Journey 1: Rider

### Goal
Find an affordable and reliable ride within the UCSB community.

### Steps

1. **Sign Up / Log In**
   - User signs up using a valid `@ucsb.edu` email.
   - User logs into the app successfully.

2. **Browse Available Rides**
   - User views a list of available rides posted by other UCSB users.
   - Rides include details such as destination, date, time, and available seats.

3. **Select a Ride**
   - User selects a ride that matches their needs.
   - User reviews ride details before joining or requesting the ride.

4. **Join or Request Ride**
   - User joins the ride or sends a request (depending on implementation).
   - Confirmation is displayed in the app.

5. **Coordinate & Complete Ride**
   - User coordinates pickup details through the app or agreed-upon communication.
   - Ride is completed successfully.

### Outcome
The rider reaches their destination affordably and safely with another UCSB affiliate.

---

## User Journey 2: Driver

### Goal
Share a ride to reduce travel costs and help other UCSB students.

### Steps

1. **Sign Up / Log In**
   - Driver logs in using a verified `@ucsb.edu` email.

2. **Create a Ride Listing**
   - Driver enters ride details:
     - Destination
     - Date and time
     - Number of available seats

3. **Publish Ride**
   - Ride becomes visible to other users in the app.

4. **Manage Riders**
   - Driver views riders who have joined or requested the ride.
   - Driver confirms participation as needed.

5. **Complete Ride**
   - Driver completes the ride with passengers.
   - Ride listing is marked as completed or expired.

### Outcome
The driver offsets travel costs while helping fellow UCSB community members.
